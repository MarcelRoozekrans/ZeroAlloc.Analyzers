# ZA0106: Avoid premature ToList/ToArray before LINQ

## Description

Detects `query.ToList().Where(...)` or `query.ToArray().Select(...)` patterns where a collection is materialized before a filtering/projection LINQ operation. This allocates a full intermediate collection unnecessarily — apply the filter first, then materialize.

## Violation

```csharp
var active = users.ToList().Where(u => u.IsActive);
var names = items.ToArray().Select(x => x.Name);
var first = results.ToList().First();
```

## Fix

```csharp
var active = users.Where(u => u.IsActive).ToList();
var names = items.Select(x => x.Name).ToArray();
var first = results.First();
```

## When to suppress

- The materialization is intentional to snapshot a changing source (e.g., concurrent collection)

## Minimum TFM

Any
