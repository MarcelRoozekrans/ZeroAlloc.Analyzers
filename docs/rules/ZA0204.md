# ZA0204: Use string.Create instead of string.Format

## Description

Detects `string.Format(...)` calls that allocate intermediate strings. Use `string.Create` with a `SpanAction<char>` to build the result directly into a buffer, avoiding the intermediate allocations.

## Violation

```csharp
var message = string.Format("Hello {0}, you have {1} items", name, count);
```

## Fix

```csharp
var message = string.Create(null, $"Hello {name}, you have {count} items");
```

Or, for full control:

```csharp
var message = string.Create(expectedLength, (name, count), static (span, state) =>
{
    "Hello ".AsSpan().CopyTo(span);
    var pos = 6;
    state.name.AsSpan().CopyTo(span.Slice(pos));
    pos += state.name.Length;
    ", you have ".AsSpan().CopyTo(span.Slice(pos));
    pos += 11;
    state.count.TryFormat(span.Slice(pos), out var written);
});
```

## When to suppress

- The format string is dynamic or loaded from a resource
- The allocation is not on a hot path

## Minimum TFM

net6.0
