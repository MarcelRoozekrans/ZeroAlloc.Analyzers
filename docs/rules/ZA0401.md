# ZA0401: Use LoggerMessage source generator

## Description

Detects `ILogger.Log*()` calls (`LogInformation`, `LogWarning`, etc.) and suggests using the `[LoggerMessage]` source generator instead. The source generator avoids boxing, string interpolation allocations, and generates optimized logging code.

## Violation

```csharp
logger.LogInformation("User {UserId} logged in", userId);
```

## Fix

```csharp
[LoggerMessage(Level = LogLevel.Information, Message = "User {UserId} logged in")]
private static partial void LogUserLoggedIn(ILogger logger, string userId);

// call site
LogUserLoggedIn(logger, userId);
```

## When to suppress

- One-off logging in non-hot code paths
- The class cannot be made `partial`

## Minimum TFM

net6.0
