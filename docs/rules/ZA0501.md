# ZA0501: Avoid boxing value types in loops

## Description

Detects value types (structs, enums, primitives) being passed to `object` or interface parameters inside loops. Each call boxes the value type, allocating on the heap on every iteration.

## Violation

```csharp
for (int i = 0; i < 100; i++)
{
    Console.WriteLine("{0}", i);       // boxes int to object
    Log(i);                            // void Log(object value) — boxes
    Format(i);                         // void Format(IFormattable f) — boxes
}
```

## Fix

```csharp
for (int i = 0; i < 100; i++)
{
    Console.WriteLine(i.ToString());   // no boxing, string is reference type
    Log(i);                            // void Log<T>(T value) — generic, no boxing
}
```

## When to suppress

- The loop has few iterations and is not in a hot path
- The boxing cost is negligible compared to other work in the loop body

## Minimum TFM

Any
