# ZA0502: Avoid closure allocations in loops

## Description

Detects lambda expressions inside loops that capture variables from the enclosing scope. Each iteration allocates a new closure (display class) object on the heap to hold the captured variables.

## Violation

```csharp
for (int i = 0; i < 100; i++)
{
    var result = items.Where(x => x > i);  // captures 'i' — allocates closure per iteration
}
```

## Fix

```csharp
for (int i = 0; i < 100; i++)
{
    int threshold = i;  // still captures, but consider restructuring
    // Option: use a loop-based approach instead of LINQ with closures
    foreach (var item in items)
    {
        if (item > threshold) { /* process */ }
    }
}
```

## When to suppress

- The loop has few iterations and is not in a hot path
- The closure capture is unavoidable and the allocation cost is negligible

## Minimum TFM

Any
