# ZA0504: Avoid defensive copies on readonly value types

## Description

Detects when a non-readonly member is invoked on a value type received as `in`, `ref readonly`, or stored in a `readonly` field. In these contexts the compiler must guarantee that the receiver is not mutated, so it silently creates a full copy of the struct before calling the member. For large structs this copy can be a significant hidden cost on hot paths.

## Violation

```csharp
struct LargeStruct
{
    public int X;
    public int GetX() => X; // not marked readonly
}

void Process(in LargeStruct s)
{
    var x = s.GetX(); // defensive copy of s created before calling GetX()
}
```

## Fix

Add the `readonly` modifier to struct methods and property getters that do not mutate state:

```csharp
struct LargeStruct
{
    public int X;
    public readonly int GetX() => X; // readonly — no defensive copy needed
}

void Process(in LargeStruct s)
{
    var x = s.GetX(); // no copy
}
```

Alternatively, declare the entire struct as `readonly struct` if no members mutate state.

## When to suppress

- The struct is small (one or two fields) and the copy cost is negligible
- The method intentionally mutates the struct and cannot be marked `readonly`

## Minimum TFM

Any
