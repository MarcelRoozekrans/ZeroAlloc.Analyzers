# ZA0601: Avoid LINQ methods in loops

## Description

Detects LINQ method calls (`.ToList()`, `.Where()`, `.Select()`, etc.) inside loops. Each call allocates iterators or collections on every iteration.

## Violation

```csharp
for (int i = 0; i < 100; i++)
{
    var filtered = items.Where(x => x > i).ToList();
}
```

## Fix

```csharp
// Cache or restructure to avoid per-iteration allocations
var filtered = new List<int>();
for (int i = 0; i < 100; i++)
{
    filtered.Clear();
    foreach (var item in items)
    {
        if (item > i) filtered.Add(item);
    }
}
```

## When to suppress

- The loop has few iterations and is not in a hot path

## Minimum TFM

Any
