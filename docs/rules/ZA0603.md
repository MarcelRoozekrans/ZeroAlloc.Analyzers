# ZA0603: Use Count/Length property instead of LINQ Count()

## Description

Detects `.Count()` LINQ extension method called on a type that already has a `.Count` or `.Length` property (e.g., `List<T>`, `ICollection<T>`, `HashSet<T>`, arrays). The LINQ method enumerates the entire collection, while the property is O(1) and allocation-free.

## Violation

```csharp
var count = myList.Count();
var len = myArray.Count();
```

## Fix

```csharp
var count = myList.Count;
var len = myArray.Length;
```

## When to suppress

- Never — the property access is strictly better in all cases.

## Minimum TFM

Any
