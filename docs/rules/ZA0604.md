# ZA0604: Use Count/Length > 0 instead of LINQ Any()

## Description

Detects `.Any()` LINQ extension method (without a predicate) called on a type that has a `.Count` or `.Length` property. Checking the property against zero is O(1) and avoids allocating an enumerator.

## Violation

```csharp
if (myList.Any())
{
    // ...
}
```

## Fix

```csharp
if (myList.Count > 0)
{
    // ...
}
```

## When to suppress

- Never — the property check is strictly better when no predicate is involved.

## Minimum TFM

Any
