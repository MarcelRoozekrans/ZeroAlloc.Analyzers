# ZA0605: Use indexer instead of LINQ First()/Last()

## Description

Detects `.First()` or `.Last()` LINQ extension methods (without a predicate) called on a type that implements `IList<T>` or is an array. Using the indexer (`[0]` or `[list.Count - 1]`) is O(1) and avoids allocating an enumerator.

## Violation

```csharp
var first = myList.First();
var last = myArray.Last();
```

## Fix

```csharp
var first = myList[0];
var last = myArray[myArray.Length - 1];
```

## When to suppress

- Never — the indexer access is strictly better for `IList<T>` and arrays.

## Minimum TFM

Any
