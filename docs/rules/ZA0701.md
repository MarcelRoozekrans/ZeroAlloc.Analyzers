# ZA0701: Use GeneratedRegex for compile-time regex

## Description

Detects `new Regex(pattern)` constructor calls with constant patterns and suggests using the `[GeneratedRegex]` source generator instead. The source generator compiles the regex at build time, avoiding runtime compilation overhead.

## Violation

```csharp
var regex = new Regex(@"\d+", RegexOptions.Compiled);
```

## Fix

```csharp
[GeneratedRegex(@"\d+")]
private static partial Regex DigitsRegex();
```

## When to suppress

- The regex pattern is dynamic (constructed at runtime)
- The class cannot be made `partial`

## Minimum TFM

net7.0
