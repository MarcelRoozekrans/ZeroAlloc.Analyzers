# ZA0803: Cache Enum.GetName/GetValues result outside loops

## Description

Detects calls to `Enum.GetName()` or `Enum.GetValues()` inside loops. These methods allocate a new string or array on every call via reflection. Cache the result in a local variable outside the loop to avoid repeated allocations.

## Violation

```csharp
for (int i = 0; i < 10; i++)
{
    var name = Enum.GetName(typeof(Color), i);
}
```

## Fix

```csharp
var values = Enum.GetValues(typeof(Color));
for (int i = 0; i < values.Length; i++)
{
    var name = Enum.GetName(typeof(Color), values.GetValue(i));
}
```

## When to suppress

- The loop body executes very few iterations and allocation cost is negligible

## Minimum TFM

Any
