# ZA0901: Consider sealing class to enable JIT devirtualization

## Description

Detects non-sealed, non-abstract classes that have no derived types within the compilation. Sealing such classes allows the JIT compiler to devirtualize virtual method calls (including `ToString`, `Equals`, `GetHashCode`, and any `virtual`/`override` members), replacing indirect calls with direct calls or even inlining them entirely.

## Violation

```csharp
class MyService
{
    public override string ToString() => "MyService";
}
```

## Fix

```csharp
sealed class MyService
{
    public override string ToString() => "MyService";
}
```

## When to suppress

- The class is intentionally designed as a base class for consumers outside the current compilation (e.g., a public extensibility point in a library)
- You plan to derive from the class in the future

## Minimum TFM

Any — devirtualization benefits apply to all modern .NET runtimes.
