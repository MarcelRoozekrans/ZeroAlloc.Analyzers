# ZA1001: Use JSON source generation instead of reflection-based serialization

## Description

Detects calls to `JsonSerializer.Serialize`, `JsonSerializer.Deserialize`, and their async variants that do not pass a `JsonTypeInfo<T>` or `JsonSerializerContext` parameter. Without source generation, `System.Text.Json` relies on runtime reflection to inspect types, which causes additional allocations and prevents trimming.

Starting with .NET 7, the JSON source generator is the recommended approach for serialization. It generates serialization metadata at compile time, eliminating reflection overhead and enabling AOT compilation.

## Violation

```csharp
var json = JsonSerializer.Serialize(myObject);
var obj = JsonSerializer.Deserialize<MyType>(json);
```

## Fix

Create a `JsonSerializerContext` and pass the generated `JsonTypeInfo<T>` to the serializer:

```csharp
[JsonSerializable(typeof(MyType))]
internal partial class MyJsonContext : JsonSerializerContext { }

var json = JsonSerializer.Serialize(myObject, MyJsonContext.Default.MyType);
var obj = JsonSerializer.Deserialize(json, MyJsonContext.Default.MyType);
```

## When to suppress

- The types being serialized are not known at compile time
- You are using polymorphic serialization that cannot be expressed via source generation
- The code path is not performance-sensitive and trimming/AOT is not required

## Minimum TFM

net7.0
