# ZA1101: Elide async/await for simple tail calls

## Description

Detects `async` methods that contain only a single `return await` statement. In this case, the `async`/`await` can be removed and the Task returned directly, avoiding the state machine allocation generated by the compiler.

## Violation

```csharp
async Task<int> GetValueAsync()
{
    return await _service.FetchAsync();
}
```

## Fix

```csharp
Task<int> GetValueAsync()
{
    return _service.FetchAsync();
}
```

## When to suppress

- The method has a `try`/`catch`/`finally` or `using` that requires the state machine for correct exception propagation
- You prefer the async stack trace information for debugging purposes
