# ZA1501: Override GetHashCode on structs used as dictionary/set keys

## Description

Detects struct types used as keys in `Dictionary<TKey, TValue>` or elements in `HashSet<T>` that do not override `GetHashCode()`. The default `ValueType.GetHashCode()` implementation uses reflection to inspect fields, which is slow and causes boxing allocations.

## Violation

```csharp
struct Point { public int X; public int Y; }

var d = new Dictionary<Point, string>();
```

## Fix

```csharp
struct Point
{
    public int X;
    public int Y;

    public override int GetHashCode() => HashCode.Combine(X, Y);
    public override bool Equals(object obj) => obj is Point p && X == p.X && Y == p.Y;
}

var d = new Dictionary<Point, string>();
```

## When to suppress

- The dictionary/set is very small and performance is not a concern
- The struct already implements `IEquatable<T>` with a custom comparer passed to the collection
